Grid system

Features

The grid system is mobile-first, implemented with flexbox. It is a 24-column and fully responsive grid system with a lot of predefined classes for layout, which could help you implement various kinds of layouts, but keep in mind that "mobile first" means you are developing websites on mobile devices by default.

container, row and col are the basic components in the grid system. container is a wrapper classes with responsive width according to your device. col must be the direct child of row.
col, col-sm, col-md, col-lg and col-xl are used for equal-width distributions with flexbox's automatic caculation of layout. In the above example, col-md will make the column equal on size on small, medium, large and extra large devices. The columns will automatically adjust their size equally when adding or removing column into the row as well as you resize the window.
Of cousrse you could use col-*, and col-*-* for your layout if the layout is known and unchanged
col-offset-* and col-offset-*-* are for column offset to the left. When using col-offset-* and col-offset-*-*, you must not set margin in CSS, since they are implemented with margin and will be overwritted.
col-push-*, col-pull-*, col-push-*-* and col-pull-*-* are used to set your column's position relative to its original default position.
We don't provide gutters between columns, since it might be any value in real websites and for present flexbox doesn't provide similar functionality. We will implement this function with CSS Grid Layout in future. If you need gutters between columns, you could set padding in columns.
col-md gutter:10%
col-md gutter:10%

To add gutter between columns, you should add customed classes with padding on column rather than add padding directly on predefined classes col-*, which will affect all columns on your website and which is not what you want.
We provide useful classes for alignment.
  • .row--vertical: change the direction
  • .row--no-wrap: unwrap the column when overflowing
  • .row--justify-center: justify the content at the center of the row
  • .row--justify-end: justify the content at the end of the row
  • .row--justify-space-between: justify the content with space-between
  • .row--justify-space-evenly: justify the content with space-evenly
  • .row--align-start: align the content at flex start
  • .row--align-center: align content at the center
  • .row--align-end: align content at the end
The responsive system is designed on the media breakpoints according to WebDimensions, which statisic the most users' devices worldwide.
  • extra small devices(xs) -- max-width: 767px
  • small devices(sm) -- min-width: 768px
  • medium devices(md) -- min-width: 992px
  • large devices(lg) -- min-width: 1200px
  • extra large devices(xl) -- min-width: 1900px

Responsive columns are designed from small devices and up, e.g, col-24 occupy 24 grids on extra small devices and up, col-sm-20 will overwrite the width on small devices and up and so on. Combining different responsive column classes are common usage in responsive web.
