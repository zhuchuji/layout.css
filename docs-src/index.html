Grid system

Overview

The grid is a mobile-first, 24-column and fully responsive grid system. We provide wrapping containers, grids, responsive utility and typical layouts with a lot of predefined classes, which could help you implement most kind of layouts (if not all). Keep in mind that "mobile first" means you are doing layouts on mobile devices by default.

Containers

We provide fixed-width and fluid-width containers. Fixed-width container has a restrict width with max-width, which changes according to media breakpoints. Fluid-width container-fluid will keep 100% wide on all devices. We add 15px padding on containers, which could be overwritten by custom classes. Even sometimes you may find out you don't need a container since row takes up 100% wide of its parent.

Grids

We build up a 24-column system with flexbox, which makes the grids flexible and magical. In the grid system row and col are the basic components in the grid system. By default, the row justify columns at the start at horizontal direction and wrap columns to newline if overflow.

We provide three kinds of columns:

  • col: column that auto grows and shrinks.
  • col-fit: column that just fit its content without growing and shrinking. You could set width for such kind of column.
  • col-*[0,24]: column with fixed width in percentage.

col must be the direct child of row. We recommend that seperate layout from your content, so be careful if you change layout style on row and col directly.

col will take up row space and make it equally distributed. Here flexbox's automatic caculation for layout makes the magic. In the above example, col will make the column equal on width. The columns will automatically adjust their size equally when adding or removing column into the row as well as you resize the window.

col-fit could make the column fit the content precisely, which will not take any extra space. You could add a width to col-fit such that it will take up fixed-width space.

Of cousrse you could use col-* if the layout is known and unchanged.

col-offset-* are for column offset to the left. When using col-offset-*, you must not set margin in CSS, since they are implemented with margin and will be overwritted.

Gutter

We don't provide gutters between columns, since it might be any value in real websites. For present, we could implement similar effect with flex justfiy-content: space-between, but it's not a fixed-width gutter. We will implement this function with CSS Grid Layout in future. If you do need gutters between columns, you could set margin or padding in columns.

col gutter: 10%
col gutter: 10%
col gutter: 10%
<div class="row basic-row">
  <div class="col block--gutter block--first">
    <div class="block--blue">col gutter: 10%</div>
  </div>
  <div class="col block--gutter">
    <div class="block--green">col gutter: 10%</div>
  </div>
  <div class="col block--gutter block--last">
    <div class="block--blue">col gutter: 10%</div>
  </div>
</div>
<style>
  .block--gutter {
    margin: 0 5%;
  }
  .block--first {
    margin-left: 0;
  }
  .block--last {
    margin-right: 0;
  }
</style>

To add gutter between columns, you should add customed classes with margin or padding on column rather than modifying directly on predefined classes col-*. Otherwise, it will affect all columns on your website and which is completely not what you want.

Responsive Column

We build the responsive grids with media query, so that it could meet the need for responsive web. Media breakpoints are set according to WebDimensions, which statisic most users' devices worldwide.

Small Medium Large Extra large
Label sm md lg xl
Media breakpoints min-width: 768px min-width: 992px min-width: 1200px min-width: 1900px

col and col-* are used for mobile layout by default. col-sm and col-sm-* are used for small device layout. col-md and col-md-* are used for medium device layout. col-lg and col-lg-* are used for large device layout and col-xl and col-xl-* are used for extra large device layout. Besides, col-offset-*, col-offset-sm-*, col-offset-md-*, col-offset-lg-*, and col-offset-xl-* are convinient.

col-24 col-sm-20 col-md-16 col-lg-12 col-xl-8 
<div class="col-24 col-sm-20 col-md-16 col-lg-12 col-xl-8">
  <div class="block--blue">
    col-24 col-sm-20 col-md-16 col-lg-12 col-xl-8
  </div>
</div>

Responsive columns are designed from small devices and up, e.g, col-24 occupy 24 grids on extra small devices and up, col-sm-20 will overwrite the width on small devices and up and so on. Combining different responsive column classes is common and effective in responsive web.

Justifying And Alignment

We provide useful classes for justifying and alignment.

  • row--horizontal: set the direction to horizontal
  • row--vertical: change the direction to vertical
  • row--wrap: wrap the columns when overflowing
  • row--no-wrap: unwrap the columns when overflowing
  • row--justify-start: justify columns at the start of the row
  • row--justify-center: justify columns at the center of the row
  • row--justify-end: justify columns at the end of the row
  • row--justify-between: justify columns with space-between
  • row--justify-around: justify columns with space-around
  • row--align-start: align columns at flex start
  • row--align-center: align columns at the center
  • row--align-end: align columns at the end

Spacing

We also provide spacing classes for slight adjustment, including margin and padding. We find them helpful thought it's easy to implement. These classes will be applied in the format ${property}${direction}-${value}.

property types:

  • m: margin
  • p: padding

direction types:

  • a: all
  • x: left and right
  • y top and bottom
  • t: top
  • b: bottom
  • l: left
  • r: right

values auto, 1, 2, 3, 4, 5, 6, 8, 10, 12, 14, 15, 16, 18, 20, 25, 30, 40, 50.

Symmetric Layout For Arabic Websites

For Arabic websites, flexbox will reverse the layout and content symetrically. We also reset asymmetric layout class e.g. col-offset-* and col-*-offset-* such that the layout do not break. Make sure you have added lang="ar" to html.
