Grid system

Overview

We create a mobile-first, 24-column and fully responsive grid system with wrapping containers, grids, responsive utility and typical layouts with a lot of predefined classes, which could help you implement most kind of layouts (if not all). It's built with flexbox, which support IE 10+. Therefore, you are at the right place if you are looking for a responsive grid system for modern browsers.

The responsive media query is based on the research on WebDimensions, which statistic most of the devices world wide. We set media breakpoints as follows.

Containers

We provide fixed-width and fluid-width containers. Fixed-width container has a restrict width with max-width, which changes according to media breakpoints. Fluid-width container-fluid will keep 100% wide on all devices. We add 15px padding on containers, which could be overwritten by custom classes. Even sometimes you may find out you don't need a container since row takes up 100% wide of its parent.

Grids

We build up the grid system with flexbox, which makes the grids flexible and magical. In the grid system, flex container and item are the two basic components. By default, the flex container justify flex items at the start in horizontal direction and wrap flex items to the newline if overflow.

We provide three kinds of flex items:

  • xs sm md lg xl: flex item that auto grows and shrinks.
  • xs-fit sm-fit md-fit lg-fix xl-fit: flex item that just fit its content without growing and shrinking. You could set width for the flex item.
  • xs-*[0,24] sm-*[0,24] md-*[0,24] lg-*[0,24] xl-*[0,24]: flex item with fixed width in percentage.

xs must be the direct child of flex, according to flex container and flex item.

xs will take up equally distributed space in flex, but they will auto self-adjust if the content change.

xs-fit could make the item fit the content precisely, which will not take up any extra space. You could add a width to col-fit such that it will take up fixed-width space.

We recommend that seperate layout from your content, so be careful if you change layouting style on flex and xs directly.

xs-offset-* are adjust item offset to the left. When using xs-offset-*, you must not set margin in CSS, since they are implemented with margin and will be overwritten.

Responsive item are designed from small devices and up, e.g., xs-24 occupy 24 grids on extra small devices and up, sm-20 will overwrite the width on small devices and up and so on. Combining different responsive column classes is a common solution and effective in responsive web.

Gutter

We don't provide gutters between item, since it might be any value in real websites. For present, we could implement similar effect with flex justfiy-content: space-between, but it's not a fixed-width gutter. We will implement this function with CSS Grid Layout in future. If you do need gutters between item, you could set margin or padding in item.

xs gutter: 10%

    <div class="block--blue">col gutter: 10%</div>

    <div class="block--green">col gutter: 10%</div>
  </div>
  <div class="col block--gutter block--last">
    <div class="block--blue">col gutter: 10%</div>

<style>
  .block--gutter {
    margin: 0 5%;
  }
  .block--first {
    margin-left: 0;
  }
  .block--last {
    margin-right: 0;
  }
</style>

To add gutter between columns, you should add customed classes with margin or padding on column rather than modifying directly on predefined classes col-*. Otherwise, it will affect all columns on your website and which is completely not what you want.

Justifying And Alignment

We provide useful classes for justifying and alignment.

  • row: set the direction to horizontal
  • column: change the direction to vertical
  • reverse: change the direction to reverse(horizontally or vertically)
  • wrap: wrap the columns when overflowing
  • no-wrap: unwrap the columns when overflowing
  • justify-content-start: justify columns at the start of the row
  • justify-content-center: justify columns at the center of the row
  • justify-content-end: justify columns at the end of the row
  • justify-content-between: justify columns with space-between
  • justify-content-around: justify columns with space-around
  • align-items-start: align columns at flex start
  • align-items-center: align columns at the center
  • align-items-end: align columns at the end

Spacing

We also provide spacing classes for slight adjustment, including margin and padding. We find them helpful thought it's easy to implement. These classes will be applied in the format ${property}${direction}-${value}.

property types:

  • m: margin
  • p: padding

direction types:

  • a: all
  • x: left and right
  • y top and bottom
  • t: top
  • b: bottom
  • l: left
  • r: right

values: auto, 1, 2, 3, 4, 5, 6, 8, 10, 12, 14, 15, 16, 18, 20, 25, 30, 40, 50.

Symmetric Layout For Arabic Websites

For Arabic websites, flexbox will reverse the layout and content symetrically. We also reset asymmetric layout class e.g. xs-offset-* such that the layout do not break. Make sure you have added lang="ar" to html.
